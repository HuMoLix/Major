# VMware 双网卡隔离网络与全流量代理测试指南 (Windows 10 客户端)

本指南介绍如何利用 VMware 虚拟网络搭建隔离测试环境，模拟 **不出网且无 Rust 编译环境的 Windows 10 虚拟机**，通过 **双网卡 Linux 网关服务端 (Ubuntu 24.04)** 加密代理出网的场景。

---

## 1. 拓扑架构与网络设计

```text
       ┌────────────────────────┐
       │   互联网 (Outer Net)    │
       └───────────┬────────────┘
                   │ 
                   │ (NAT / Bridge 模式)
                   ▼
  ┌─────────────────────────────────┐
  │   VM 2: 双网卡 Ubuntu 24.04 网关 │
  │  - 网卡 1 (ens33): 192.168.3.219  │ ──> 出网网卡
  │  - 网卡 2 (ens38): 192.168.100.1  │ ──> 仅主机隔离网卡
  └────────────────┬────────────────┘
                   │
                   │ (VMnet2 仅主机局域网)
                   ▼
  ┌─────────────────────────────────┐
  │     VM 1: 不出网 Windows 10      │
  │  - 网卡 1: 192.168.100.2          │ ──> 隔离局域网 (无法直接访问外网)
  └─────────────────────────────────┘
```

---

## 2. 第一步：VMware 虚拟网络配置 (VMnet2)

1. 在 VMware 菜单栏选择 **编辑 -> 虚拟网络编辑器 (Virtual Network Editor)**。
2. 点击右下角 **更改设置** 获取管理员权限。
3. 点击 **添加网络**，选择 `VMnet2`。
4. 将 `VMnet2` 配置为 **仅主机模式 (Host-Only)**。
5. **取消勾选** “使用本地 DHCP 服务将 IP 地址分配给虚拟机”。
6. 将子网 IP 设置为 `192.168.100.0`，子网掩码为 `255.255.255.0`。
7. 点击确定保存。

---

## 3. 第二步：配置虚拟机网卡与系统 IP

### 1. 配置 VM 1 (不出网 Windows 10 虚拟机)
1. 在 VMware 设置中，将 Windows 10 的网卡网络连接选择为 **自定义 -> VMnet2**。
2. 启动 Windows 10 虚拟机，打开“网络和共享中心” -> “更改适配器设置”。
3. 右键点击网卡属性 -> 选择“Internet 协议版本 4 (TCP/IPv4)”。
4. 配置静态 IP 地址，且**不要填写默认网关和 DNS**（确保其无法直接出网）：
   - **IP 地址**：`192.168.100.2`
   - **子网掩码**：`255.255.255.0`
   - **默认网关**：*留空*
   - **DNS 服务器**：*留空*
5. 在 Windows CMD 中运行 `ping 8.8.8.8`，应当报错 `传输失败。常见故障。` 或超时，代表已经彻底断网。

### 2. 配置 VM 2 (双网卡 Ubuntu 网关/服务端)
1. 在 VMware 设置中，为该虚拟机添加两块网卡：
   - **网卡 1**：选择 **NAT** 模式（用于访问互联网，假设系统内网卡名为 `ens33`）。
   - **网卡 2**：选择 **自定义 -> VMnet2**（用于连接隔离局域网，假设网卡名为 `ens38`）。
2. 启动虚拟机，配置静态 IP：
   ```bash
   sudo ip addr add 192.168.100.1/24 dev ens38
   sudo ip link set dev ens38 up
   ```
3. 验证连通性：在 Windows 10 虚拟机上运行 `ping 192.168.100.1` 应当能够正常 Ping 通。

---

## 4. 第三步：配置并启动 Linux 服务端 (VM 2)

1. **自动安装配置 WireGuard**：
   在 Ubuntu 虚拟机中执行配置脚本：
   ```bash
   sudo ./setup_wireguard.sh
   ```
   *说明：脚本会自动识别外网网口 `ens33` 并建立 NAT 路由伪装规则。*
2. **配置并启动 API 服务**：
   编辑 [main.py](file:///E:/major/server/main.py)：
   - 将 `SERVER_ENDPOINT` 设为内网监听的接口：`"192.168.100.1:51820"`。
   - 启动服务：`uvicorn main:app --host 0.0.0.0 --port 8000`
3. **生成激活码**：
   ```bash
   .venv/bin/python generate_key.py --days 30
   # 复制记录生成的激活码，如 KEY-JNS4-450J-6824-L7SR
   ```

---

## 5. 第四步：编译并拷贝客户端程序到 Windows 10

1. **在有 Rust 编译环境的主机上编译 Release 版本**：
   打开您主机的命令行工具，进入项目客户端目录执行编译：
   ```powershell
   cd E:\major\client
   cargo build --release
   ```
2. **收集客户端所需文件**：
   编译完成后，在以下目录收集这两个文件：
   - 客户端执行程序：`E:\major\client\target\release\client.exe`
   - 网卡驱动：`E:\major\client\wintun.dll`
3. **拷贝至 Windows 10 虚拟机**：
   通过 VMware 共享文件夹、U盘镜像或直接复制粘帖的方式，将上述 `client.exe` 和 `wintun.dll` 两个文件拷贝到 Windows 10 虚拟机内的同一个文件夹中（例如 `C:\VPNClient\`）。

---

## 6. 第五步：运行客户端与 Windows 路由代理设置

1. 在 Windows 10 虚拟机中，**以管理员身份打开 CMD 或 PowerShell**。
2. 进入客户端放置目录：
   ```cmd
   cd C:\VPNClient
   ```
3. 启动客户端：
   ```cmd
   client.exe
   ```
4. 输入激活码激活连接。网卡创建成功后，控制台会显示已分配的隧道 IP（假设为 `10.0.0.4`）。
5. **设置全流量代理路由**（由于没有默认网关，我们需要在 Windows 上配置全局路由走 VPN）：
   在 Windows 10 虚拟机的**管理员 CMD 窗口**中执行以下命令：
   ```cmd
   # 1. 确保到网关服务器 IP 的流量依然走原来的局域网网口 (防止流量环路套娃)
   # 格式: route add <网关IP> <本机静态IP>
   route add 192.168.100.1 192.168.100.2
   
   # 2. 将全局默认网关指向虚拟网口 (IP 需要替换为实际分配的客户端 Tunnel IP)
   # 格式: route add 0.0.0.0 mask 0.0.0.0 <分配的客户端IP>
   route add 0.0.0.0 mask 0.0.0.0 10.0.0.4 metric 1
   ```

---

## 7. 验证 Windows 10 代理出网

在 Windows 10 虚拟机的 CMD 中，尝试访问外网：
```cmd
ping 8.8.8.8
```
或者在浏览器中打开网页（例如配置临时公网 DNS：`8.8.8.8`）。

如果能够成功 Ping 通并获取网页，说明**原本处于彻底断网状态的 Windows 10 虚拟机，已成功通过内存解密的 Wintun 驱动与 Ubuntu 网关建立了安全隧道，并成功实现了全局代理出网！**
测试完毕后，在 CMD 窗口中按 `Ctrl+C` 退出客户端，路由和网卡会自动注销。
